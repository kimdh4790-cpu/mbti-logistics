---

### 3-1. 기존 기술의 문제점

쿠팡 등 물류 플랫폼에서 대리점에 지급하는 정산 데이터는 **기사 아이디(ID) 단위**로 제공된다.

현실에서 발생하는 문제:
1. **아이디 지원**: 기사 A가 사정이 있어 기사 B의 아이디로 배송 운행하는 경우 발생
2. **수작업 분리**: 기존에는 관리자가 엑셀을 수동으로 확인하여 실제 운행자에게 금액 분배
3. **오류 발생**: 날짜별 건수 추적이 안 되어 계산 오류, 분쟁 발생
4. **시간 낭비**: 기사 30명 기준 월 5~10시간 수작업

---

### 3-2. 본 발명의 핵심 기술 (2026-05-31 구현 완성)

#### ① 날짜별 라우트 건수 추적 (dateRoutes)

엑셀 파싱 시 기사 아이디별로 **날짜×라우트** 조합의 건수를 누적 저장한다.

```
drvMap[기사ID].dateRoutes = {
  "2026-04-29": {
    "213B": { cnt: 44, ret: 0 },
    "213C": { cnt: 76, ret: 0 },
    "213D": { cnt: 76, ret: 0 }
  },
  "2026-05-16": {
    "109C": { cnt: 39, ret: 0 },
    "109D": { cnt: 52, ret: 0 }
  }
}
```

#### ② 아이디 지원 규칙 등록

관리자가 지원 규칙을 등록한다:

| 필드 | 내용 | 예시 |
|------|------|------|
| fromId | 아이디를 빌려준 기사 | kycandksh |
| toId | 실제 운행한 기사 | lifeisegg6 |
| dates | 지원한 날짜 목록 | ["2026-04-29"] |
| routes | 지원한 라우트 (없으면 전체) | [] |

#### ③ 자동 정산 분배 로직

1. `fromId`의 `dateRoutes`에서 지정 날짜·라우트 건수 추출
2. 해당 건수 × 단가 → `toId`(실제 운행자)의 기타(+)에 자동 가산
3. `fromId`의 기타(-)에 동일 금액 차감
4. `fromId`의 라우트별 실적에서 건수 차감

#### ④ XLSX.js 날짜 파싱 버그 보정

- 문제: XLSX.js가 `2026-04-29 00:00 KST`를 `2026-04-25 23:59:08`로 파싱
- 해결: 시간이 23:58 이상이면 +2분 보정하여 정확한 날짜 계산
